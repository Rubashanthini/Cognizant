# LibraryManagement

Cognizant Digital Nurture 5.0 — Deep Skilling Engineering Concepts.
A single Maven project that implements **all 9 Spring Framework / Spring
Boot exercises** for a Library Management application.

---

## 1. Project Setup

### Prerequisites
- JDK 8 (or higher, project is compiled targeting Java 8)
- Maven 3.6+
- An IDE such as IntelliJ IDEA, Eclipse, or VS Code (optional — the project
  also runs fully from the command line)

### Importing the project
1. Unzip / clone the `LibraryManagement` folder.
2. Open it in your IDE as an **existing Maven project** (IntelliJ:
   `File > Open` and select the folder containing `pom.xml`; Eclipse:
   `File > Import > Existing Maven Projects`).
3. Let the IDE download dependencies (or run `mvn clean install` from the
   command line — see below).

### Maven commands
```bash
# Compile the project
mvn clean compile

# Run unit tests
mvn test

# Build the full project (compiles, tests, packages into target/LibraryManagement.jar)
mvn clean install

# Skip tests if you just want the jar quickly
mvn clean package -DskipTests
```

## 3. How each Exercise is implemented

| Exercise | What it asks for | Where it lives |
|---|---|---|
| 1 | Spring Core config, XML config, bean definitions | `applicationContext.xml`, `BookService`, `BookRepository` |
| 2 | Dependency Injection, Setter Injection, IoC | `BookService.setBookRepository()`, wired via `<property>` in `applicationContext.xml` |
| 3 | Spring AOP, Logging Aspect, execution time logging | `LoggingAspect.logExecutionTime()` (`@Around`) |
| 4 | Maven config, Java 8 compiler plugin, Spring Context/AOP/WebMVC deps | `pom.xml` |
| 5 | Spring IoC container, XML bean configuration | `applicationContext.xml` |
| 6 | Annotation config: `@Component`/`@Repository`/`@Service`, component-scan | `applicationContext-annotation.xml`, `@Service`/`@Repository` on classes, `AnnotationConfig.java` (Java-config alternative) |
| 7 | Constructor Injection + Setter Injection | `BookService` constructor & setter, both wired in `applicationContext.xml` (`constructor-arg` + `property`) |
| 8 | Spring AOP, Before/After advice, AspectJ auto-proxy | `LoggingAspect.logBefore()` / `logAfter()`, `<aop:aspectj-autoproxy/>` |
| 9 | Spring Boot, Spring Data JPA, H2, REST CRUD | `LibraryManagementApplication`, `Book` entity, `BookJpaRepository`, `BookController` |

### A note on naming (`BookRepository` vs `BookJpaRepository`)
Exercises 1–8 ask for a plain **class** named `BookRepository` (in-memory,
used to teach IoC/DI/AOP). Exercise 9 separately asks for a Spring Data JPA
**interface** also called `BookRepository`. Java does not allow two
different types with the same name in the same package, so the Spring Data
JPA interface used for Exercise 9 has been named **`BookJpaRepository`**
instead — it plays exactly the role Exercise 9 describes (a Spring Data
JPA repository for the `Book` entity, backed by H2). This is documented in
the Javadoc of both classes.

## 4. Running the Spring Core / AOP demos (Exercises 1, 2, 3, 5, 7, 8)

These use a classic `ClassPathXmlApplicationContext` — no web server, no
database, just the Spring IoC container.

**From your IDE:** run `com.library.application.SpringCoreMainApp` (explicit
XML `<bean>` definitions, constructor + setter injection, AOP logging).

**From the command line:**
```bash
mvn compile exec:java -Dexec.mainClass="com.library.application.SpringCoreMainApp"
```
> If the `exec` goal is not available, simply run the class from your IDE,
> or build the jar and run it with the main class on the classpath.

## 6. Running the Spring Boot REST API (Exercise 9)

```bash
mvn spring-boot:run
```
or, after `mvn clean package`:
```bash
java -jar target/LibraryManagement.jar
```

The application starts an embedded Tomcat server on **port 8080**, creates
the `book` table from `schema.sql`, and loads sample rows from `data.sql`.

### API testing

You can use `curl`, Postman, or the browser.

**Get all books**
```bash
curl http://localhost:8080/books
```

**Get a book by id**
```bash
curl http://localhost:8080/books/1
```

**Create a new book**
```bash
curl -X POST http://localhost:8080/books \
  -H "Content-Type: application/json" \
  -d '{"title":"Design Patterns","author":"Gang of Four","isbn":"978-0201633610","price":54.99,"publishedYear":1994}'
```

**Update a book**
```bash
curl -X PUT http://localhost:8080/books/1 \
  -H "Content-Type: application/json" \
  -d '{"title":"Effective Java (3rd Edition)","author":"Joshua Bloch","isbn":"978-0134685991","price":41.99,"publishedYear":2018}'
```

**Delete a book**
```bash
curl -X DELETE http://localhost:8080/books/1
```

### Expected output
- `GET /books` → `200 OK` with a JSON array of books (4 sample books preloaded).
- `GET /books/{id}` → `200 OK` with the book JSON, or `404 Not Found` if the id doesn't exist.
- `POST /books` → `201 Created` with the newly created book, including its generated `id`.
- `PUT /books/{id}` → `200 OK` with the updated book, or `404 Not Found`.
- `DELETE /books/{id}` → `204 No Content`, or `404 Not Found`.

### H2 console (optional)
Visit `http://localhost:8080/h2-console` while the app is running.
JDBC URL: `jdbc:h2:mem:librarydb`, username `sa`, empty password.

## 7. Summary of technologies used

- Spring Core / IoC container (XML and annotation-based)
- Spring Dependency Injection (constructor and setter)
- Spring AOP + AspectJ (`@Aspect`, `@Before`, `@After`, `@Around`, `aspectj-autoproxy`)
- Spring Boot 2.7.x
- Spring Data JPA
- H2 in-memory database
- Spring WebMVC / REST controllers
- Maven (Java 8 compiler plugin, Spring Boot Maven plugin)
